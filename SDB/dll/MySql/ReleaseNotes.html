Release Notes

Version 0.76 - 3/12/04

  • Renamed assembly to ByteFX.MySqlClient.dll
  • Moved all design components into MySqlClient.Design
  • Supporting 2,4, & 8 byte integer lengths from MySQL
  • Better support for compression using the CompressedStream class
  • Pooling now using the PING command to determine if MySQL is alive
  • Fields now read values directly out of the rowset buffer. This should speed up operations on multiple rows with blobs
  • Fixed bug where connecting to a down or non-existent server would not timeout
  • Improved connection string handling. Now if there is a problem with the connection string, then the existing connection string is not changed.
  • Fixed connection object where an exception throw prior to the connection opening would not leave the connection in the connecting state
  • Added GUID support
  • Fixed sequence out of order bug (thanks Mark Reay)
  • And many more improvements that I can't think of right now....

Version 0.75 - 1/27/04

  • MySQL year datatype is now supported and handled internally as a 32 bit integer
  • Compression is now working again!!!  Use compress=true on your connection string to enable this
  • Parameters with values that were of type System.TimeSpan or System.Enum are now handled correctly
  • Improved connection string handling.  Specifically, defaults are handled better and the behavior more closely matches that of SqlClient.  If you specify a connection string that has a syntax error, the old connection string values should not be overwritten.
  • A MySqlException is now thrown if a user id is not included in the connection string. 
  • Localhost is now used if server is not specified on the connection string
  • Leaving password= out of the connection string now works properly
  • A MySqlException is now thrown if an attempt is made to set the connection string while the connection is open.
  • Removed MultiHostStream and MySqlStream and replaced it with Common/StreamCreator that creates the appropriate stream based on inputs. 
  • Speed is greatly improved due to better handling of network connection.
  • A pipe connection is now used when you specify Use Pipe=true on the connection string.  Note that this replaces the older method of selecting pipes by specifying -1 as the port.
  • Fixed some pooling bugs!!  The provider should recover better from closed or timed out connections.
  • Much more testing!!
  • MySql 4.1 style timestamps are now handled
  • MySqlException now includes the error number returned from the server
  • Text blob columns are now returned as string data types.
